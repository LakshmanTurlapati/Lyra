Module docstring: "eval_runner.py -- Unified evaluation runner invoking standard benchmark suites (D-01)."

CLI interface (per D-01):
```
python3 -m scripts.eval_runner --model HuggingFaceTB/SmolLM2-1.7B-Instruct \
  --benchmarks tool-calling,code,knowledge \
  --output results/smollm2_base.json \
  --config configs/eval.yaml
```

Arguments via argparse:
- `--model` (str, required): HuggingFace model path or local path
- `--benchmarks` (str, required): Comma-separated list from {tool-calling, code, knowledge}
- `--output` (Path, required): Output JSON file path
- `--config` (Path, default=Path("configs/eval.yaml")): Eval config path
- `--device` (str, optional): Override auto-detected device ("mps" or "cpu")

Functions to implement:

1. `detect_device() -> str`:
   - Import torch
   - If torch.backends.mps.is_available(): set os.environ["PYTORCH_ENABLE_MPS_FALLBACK"] = "1", return "mps"
   - Else: return "cpu"
   - Per D-02: NO CUDA support

2. `run_knowledge_benchmarks(model_path: str, device: str, config: EvalConfig) -> CategoryResult`:
   - Import lm_eval
   - Call lm_eval.simple_evaluate(model="hf", model_args=f"pretrained={model_path},dtype={config.dtype}", tasks=config.knowledge_tasks, device=device, batch_size=config.batch_size, num_fewshot=config.num_fewshot.get("mmlu", 5))
   - Note: lm-eval-harness does NOT support per-task num_fewshot in simple_evaluate -- use the first task's value as the global setting. Document this limitation.
   - Extract results: results["results"][task_name] contains metric keys like "acc,none" or "acc_norm,none"
   - Map to BenchmarkResult objects: mmlu uses "acc", arc_challenge uses "acc_norm", hellaswag uses "acc_norm"
   - Return CategoryResult(category="knowledge", benchmarks=[...])

3. `run_code_benchmarks(model_path: str, config: EvalConfig) -> CategoryResult`:
   - For each dataset in config.code_datasets:
     - cmd = ["evalplus.evaluate", "--model", model_path, "--dataset", dataset, "--backend", "hf", "--greedy"]
     - subprocess.run(cmd, capture_output=True, text=True, check=False)
     - Parse stdout for pass@1 score (look for "pass@1" in output text)
   - Return CategoryResult(category="code", benchmarks=[...])
   - Use subprocess list form (no shell=True) per security threat T-03-05

4. `run_tool_calling_benchmarks(model_path: str, device: str, config: EvalConfig) -> CategoryResult`:
   - Phase 1: Generate responses using transformers (MPS-compatible)
     - Load model with AutoModelForCausalLM.from_pretrained(model_path, torch_dtype=torch.float32).to(device)
     - Load tokenizer with AutoTokenizer.from_pretrained(model_path)
     - Load BFCL test prompts from bfcl-eval package data
     - Generate responses and write to BFCL-expected directory structure
   - Phase 2: Evaluate using BFCL AST checker
     - Set os.environ["BFCL_PROJECT_ROOT"] to results/bfcl directory
     - cmd = ["bfcl", "evaluate", "--model", model_name, "--test-category"] + config.bfcl_test_categories
     - subprocess.run(cmd, capture_output=True, text=True, check=False)
     - Parse BFCL score output
   - Return CategoryResult(category="tool-calling", benchmarks=[...])
   - Use subprocess list form (no shell=True) per security threat T-03-05

5. `format_summary_table(result: EvalResult) -> str`:
   - Print header: Category, Benchmark, Metric, Score
   - Print separator line with dashes
   - For each category, for each benchmark: format row with fixed column widths
   - Use f-string formatting, NO external libraries
   - NO emoji characters

6. `main() -> int`:
   - Parse CLI args
   - Load config via load_eval_config(args.config)
   - Detect or use provided device
   - Parse benchmarks from comma-separated string
   - Run requested benchmark suites
   - Build EvalResult with model_name extracted from model_path (last path component), timestamp as datetime.now().isoformat(), device
   - Write EvalResult JSON to args.output (create parent dirs with Path.mkdir(parents=True, exist_ok=True))
   - Print summary table to stdout (per D-08)
   - Return 0 on success, 1 on error

| Boundary | Description |

| User CLI args -> eval_runner.py | Model path and benchmark names from command line |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | subprocess commands in eval_runner.py | mitigate | Use list-form subprocess.run (never shell=True); validate model name contains only alphanumeric, hyphen, underscore, forward slash characters before passing to subprocess |
| T-03-06 | Elevation of Privilege | evalplus code execution | mitigate | Do not disable evalplus sandboxing; document that eval runs model-generated code in sandbox |
| T-03-07 | Tampering | Model path argument | mitigate | Validate model_path is either a valid local Path that exists or matches HuggingFace model ID pattern (org/model format) before passing to library functions |
| T-03-08 | Denial of Service | Benchmark timeout | accept | Standard benchmarks may take hours on CPU for large task sets; user accepts this by running the command. No timeout enforcement needed for local dev tool. |
| T-03-09 | Information Disclosure | Model weights downloaded to local cache | accept | Standard HuggingFace behavior; model weights are public (SmolLM2-1.7B is open-source); no credential leakage |
</threat_model>
