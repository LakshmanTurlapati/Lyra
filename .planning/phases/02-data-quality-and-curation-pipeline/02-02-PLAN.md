    Key functions:

    1. load_config(config_path: Path) -> PipelineConfig: Open YAML file, yaml.safe_load(), validate with PipelineConfig.model_validate(). Return typed config.

    2. get_domain_config(config: PipelineConfig, domain: str) -> dict: Merge config.defaults with config.domains.get(domain). Return as plain dict for passing to scorer/dedup/style modules. Deep-merge the "style" sub-dict. Use .model_dump() for serialization.

    3. run_pipeline(input_path: Path, output_path: Path, config: PipelineConfig, domain: str) -> dict: Execute 4 stages sequentially:
       - Stage 1 (Format): For each line in input JSONL, parse JSON, validate via Conversation.model_validate(). Collect valid samples, log rejected with reason.
       - Stage 2 (Quality): For each valid sample, call score_sample(sample, domain_config_dict). Attach result as sample["_quality"]. Keep only samples where _quality["pass"] is True.
       - Stage 3 (Dedup): Call deduplicate_batch(scored_samples, domain_config_dict) with ngram_size, dedup_threshold, dedup_scope from config.
       - Stage 4 (Style): For each deduped sample, call validate_style(sample, domain, domain_config_dict). Keep only passing samples.
       - Write final samples to output_path as JSONL (one JSON object per line).
       - Return stats dict: {"input_count": int, "format_valid": int, "quality_pass": int, "after_dedup": int, "after_style": int, "output_count": int}

       - --config: Path to pipeline.yaml (default: configs/pipeline.yaml)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSONL file -> pipeline | Raw conversation files from disk parsed line-by-line |
| pipeline.yaml -> pipeline | User-editable config controls all thresholds and paths |
| CLI arguments -> pipeline | User provides file paths and domain via argparse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | scripts/curate_pipeline.py | mitigate | Use yaml.safe_load() exclusively for config loading -- never yaml.load() which allows arbitrary code execution |
| T-02-05 | Tampering | scripts/curate_pipeline.py | mitigate | Validate config with Pydantic PipelineConfig model before use -- rejects unexpected fields and invalid value ranges |
| T-02-06 | Information Disclosure | configs/pipeline.yaml | accept | Config contains only thresholds and paths, no secrets. Committed to repo intentionally. |
| T-02-07 | Denial of Service | scripts/curate_pipeline.py | mitigate | Process JSONL line-by-line for format validation (Stage 1) to avoid loading entire file into memory. Batch dedup is bounded by expected 5K sample limit. |
| T-02-08 | Tampering | CLI path arguments | mitigate | Use pathlib.Path for all file operations; argparse validates required arguments; output directory created with parents=True but no symlink following |
</threat_model>
