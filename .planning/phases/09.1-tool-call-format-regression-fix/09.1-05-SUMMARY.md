# Phase 09.1 Plan 05: Retrain Lyra Model Summary

**Retrained Lyra on the scaled assembled dataset with the template persistence fix baked into the merge step.**

## Performance

- **Smoke test:** 10 steps, completed successfully in 1m 15s
- **Full training:** 1,496 steps, completed in 3h 37m 15s
- **Epoch 1 eval_loss:** 0.1613
- **Epoch 2 eval_loss:** 0.1384
- **Hardware:** MPS, single training process

## Dataset Used

Plan 05 originally referred to a 25K-scale dataset. The completed Plan 04 curation output is the source of truth:

| Split | Total | Tool-calling | Code | Knowledge |
|-------|-------|--------------|------|-----------|
| Train | 11,963 | 10,793 | 584 | 586 |
| Validation | 666 | 600 | 33 | 33 |
| Test | 663 | 599 | 32 | 32 |
| **Total** | **13,292** | **11,992** | **649** | **651** |

The tool-calling domain, which is the primary target for this regression fix, remained above the Plan 04 minimum. Code and knowledge were below the original target because full-scope dedup removed template-pool duplicates.

## Deviations from Plan

- The training dataset was 13,292 samples, not 25K. This follows Plan 04's actual curation output and is documented there as a dedup/template-diversity limitation.
- The 10-step smoke test only logged one loss point because `scripts/train.py` logs every 10 steps. The full training run later confirmed loss decreased from 1.5705 at step 10 to the low 0.1-0.2 range in epoch 2.
