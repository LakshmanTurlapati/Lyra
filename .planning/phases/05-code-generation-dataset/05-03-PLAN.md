<objective>
Run the existing curation pipeline on all raw code generation batches to filter, deduplicate, score, and style-validate the data. Then verify sample quality through a human checkpoint.

Purpose: The 2x overgeneration strategy produces ~3,400 raw samples that need to be filtered down to ~1,667 high-quality samples. The curation pipeline from Phase 2 handles this automatically using format validation, quality scoring, n-gram Jaccard dedup, and code-domain style validation (require_code_blocks, max_prose_ratio <= 0.4). The human checkpoint ensures the curated output meets quality expectations before feeding into Phase 7 assembly.

Output: `datasets/code/curated/code-curated.jsonl` containing ~1,667 curated samples ready for dataset assembly.
</objective>
