**Critical implementation details:**

1. **Query diversity:** Each category has 80-150+ unique query templates with `{placeholder}` variables. Placeholders are filled from domain-specific value pools (city names, file paths, package names, git commands, etc.). This prevents repetitive user messages that would trigger dedup.

2. **Response diversity:** `generate_tool_response()` varies response structures: sometimes flat JSON, sometimes nested, sometimes arrays, sometimes plain text. Error responses use varied formats (HTTP-style `{"error": "...", "code": 404}`, simple strings, structured error objects). This avoids Pitfall 2 (monotonic responses).

3. **Edge case allocation per D-05/D-06:** Each batch generator takes `count` and allocates ~25% as edge cases. The 4 edge case types are rotated through:
   - No-tool-needed: tools defined in `tools` array but assistant message has `content` only, no `tool_calls`
   - Error handling: tool response contains error object, assistant message explains the error gracefully
   - Ambiguous: some samples call the tool, some answer directly for similar queries
   - Parameter edge cases: optional params omitted, empty string values, boolean params

4. **System prompt loading:** Loads from `templates/system-prompts.yaml` via yaml.safe_load. tool_assistant for single-call/multi-turn/parallel, mcp_assistant for MCP, cli_assistant for CLI.

5. **Schema pool usage:** `get_tools_for_category()` selects 1-3 tools from the pool appropriate for the category. Single-call gets 1-3 tools, CLI gets run_command (+ optionally run_command_in_dir), multi-turn gets 2-3 tools, parallel gets 2-3 of the same or different tools, MCP gets mcp_meta tools plus 1-2 discovered tools.

6. **Seeded randomness:** Each batch function accepts a `seed` parameter for reproducibility. `random.Random(seed)` instance used per batch (not global state).

7. **Per-sample validation:** `validate_batch()` runs `Conversation.model_validate()` on every sample before writing. Any sample that fails validation is logged and excluded.

8. **CLI interface per D-08:**
   ```
   python -m scripts.generate_tool_data --category single-call --count 50 --batch 1
   # Writes: datasets/tool-calling/single-call-batch-01.jsonl
   ```
