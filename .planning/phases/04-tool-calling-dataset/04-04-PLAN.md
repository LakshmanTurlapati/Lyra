**Step 3: Run the full curation pipeline.**

```bash
python -m scripts.curate_pipeline \
    --input datasets/tool-calling/curated/tool-calling-raw-merged.jsonl \
    --domain tool-calling \
    --config configs/pipeline.yaml \
    --output datasets/tool-calling/curated/tool-calling-curated.jsonl
```

The pipeline runs 4 stages:
1. Format validation (via validate_format.py Conversation model)
2. Quality scoring (via quality_scorer.py -- format compliance, completeness, naturalness, diversity)
3. Deduplication (via dedup.py -- n-gram Jaccard with response-scope per Phase 2 decision)
4. Style validation (via style_validator.py -- tool-calling domain: max_tokens=800, require_tool_calls=true, allow_terse=true)

Note: `require_tool_calls=true` in the style config means samples WITHOUT tool_calls will be flagged by the style validator. However, our ~25% edge case samples intentionally lack tool_calls. This is expected -- the style validator filters them, but we deliberately overgenerate by 2x (3,300 raw -> 1,667 target) to account for this filtering.

**Step 5: Analyze category distribution in curated output.**

```python
import json
from collections import Counter

category_counts = Counter()
with open('datasets/tool-calling/curated/tool-calling-curated.jsonl') as f:
    for line in f:
        conv = json.loads(line)
        msgs = conv['messages']
        # Detect category from structure
        system_content = msgs[0].get('content', '')
        has_tool_calls = any(m.get('tool_calls') for m in msgs)
        tool_call_rounds = sum(1 for m in msgs if m.get('tool_calls'))
        tool_names = [tc['function']['name'] for m in msgs if m.get('tool_calls') for tc in m['tool_calls']]
        max_parallel = max((len(m['tool_calls']) for m in msgs if m.get('tool_calls')), default=0)

        if 'mcp_list_servers' in tool_names or 'mcp_list_tools' in tool_names:
            category_counts['mcp'] += 1
        elif 'run_command' in tool_names or 'run_command_in_dir' in tool_names:
            category_counts['cli'] += 1
        elif max_parallel >= 2:
            category_counts['parallel'] += 1
        elif tool_call_rounds >= 2:
            category_counts['multi-turn'] += 1
        elif has_tool_calls:
            category_counts['single-call'] += 1
        else:
            category_counts['edge-case-no-tool'] += 1

total = sum(category_counts.values())
for cat, count in sorted(category_counts.items()):
    print(f'{cat}: {count} ({count/total*100:.1f}%)')
print(f'Total: {total}')
```

Distribution targets per D-02 (with ~20% tolerance):
- single-call: 25-45% of curated
- cli: 15-35% of curated
- multi-turn: 8-22% of curated
- parallel: 8-22% of curated
- mcp: 5-15% of curated

If any category is severely underrepresented (below its lower bound), regenerate additional batches for that category and re-run the pipeline.
